# `talks.json` schema

Источник истины для архива видео-выступлений. Читается `scripts/archive-talks.py`.

## Поля

Каждая запись в массиве `talks`:

| поле | обязат. | описание |
|---|---|---|
| `id` | да | Уникальный slug (kebab-case). Используется как имя папки в `archive/videos/<id>/`. Не меняйте после первого скачивания. |
| `title` | да | Человекочитаемое название. |
| `description` | нет | Короткое описание. Попадёт в schema.org VideoObject, если будем подключать. |
| `category` | нет | `talk` / `podcast` / `interview` / `making-of` и т.п. Для будущей группировки. |
| `lang` | нет | Основной язык (`ru` / `en`). |
| `date` | нет | Дата выступления `YYYY-MM-DD`. Если пусто — `yt-dlp` вытащит `upload_date` из `*.info.json`. |
| `sources` | да | Массив URL. `yt-dlp` пробует по очереди до первого успешного — это делает архив устойчивым к удалению с конкретной площадки. Поддерживается всё, что поддерживает yt-dlp: YouTube, VK, Rutube, Vimeo, Dzen и т.д. |

## Как добавить новое видео

1. Допишите запись в `talks.json`.
2. Запустите `./scripts/archive-talks.py` (нужен `yt-dlp`: `brew install yt-dlp`).
3. Новые видео скачаются, уже скачанные пропустятся (маркер `.done` в папке).

## Что сохраняется в `archive/videos/<id>/`

- `*.mp4` — видео максимального качества
- `*.info.json` — метаданные (название, автор, дата загрузки, длительность, теги)
- `*.description` — текст описания с площадки
- `*.jpg` / `*.webp` — thumbnail
- `*.vtt` — субтитры (ручные + авто, где есть, ru/en)
- `.done` — маркер успешного скачивания (для идемпотентности)
